Service Policies

Here is a list of all Policies defined for this Service. These Policies will be applied to all invocations of this Service by any Application, regardless of the Plan used in its Contract.

Default policies applies for each service implicitly:

  • CORS - Default values are enabled for each published API. Default headers are: "Accept,Accept-Version,Content-Length,Content-MD5,Content-Type,Date,apikey,Authorization".

    INFO: The marketplace's try-out functionality will not work correctly if your service makes use of custom headers not declared in the CORS policy.

  • Key Authentication - Default enabled for all published APIs, client API keys are provided upon subscription
  • HTTP Logging - Default enabled. All transaction information for a published API will be aggregated in the API metrics module.
  • ACL - Access Control List.

    WARNING: Without an ACL policy in place, any application with valid credentials will be able utilize your service, regardless of the presence of contracts
